# Plan: Add Quarm Server Protocol Support to showeq-daemon

## Context

The user wants to create a worktree from `showeq-daemon` that supports the Quarm server protocol. Quarm is a server implementation for EverQuest that uses a different protocol than standard EQEmu servers.

### Key Findings

**EQMacEmu Protocol Structure:**
- Located in `EQMacEmu/common/` - this is the source of truth for the EQEmu protocol
- Uses `emu_opcodes.h` and `emu_oplist.h` for opcode definitions
- Key files:
  - `common/emu_opcodes.h` - defines the `EmuOpcode` enum via preprocessor include of `emu_oplist.h`
  - `common/emu_oplist.h` - contains the opcode list (N() macro entries)
  - `common/eq_packet.h` - defines `EQPacket`, `EQProtocolPacket`, `EQApplicationPacket`, `EQRawApplicationPacket`
  - `common/eq_packet_structs.h` - packet struct definitions
  - `common/quarm_item_data.h` - Quarm-specific item data (self-found tracking)

**showeq-daemon Protocol Structure:**
- Located in `showeq-daemon/src/` - older EQ protocol implementation
- Uses `opcodetable.h` (auto-generated by DumpOpcodeTable.idc from EQMacEmu sources!)
- Key files:
  - `src/emu_opcodes.h` - defines `EmuOpcode` enum (older version from EQMacEmu)
  - `src/packetstream.cpp` - handles packet processing, session tracking, ARQ (acknowledgment/retransmission)
  - `src/packet.h`/`src/packetcommon.h` - packet types and utilities
  - `src/sessionadapter.h`/`src/sessionadapter.cpp` - WebSocket client bridge (newer)
  - `src/seq/v1/events.proto` - protobuf schema for streaming to clients

**Protocol Differences (Quarm vs EQMacEmu):**
- Quarm is a fork/mod of EQMacEmu with specific customizations
- The protocol parsing structure is largely compatible
- Key Quarm-specific features in EQMacEmu:
  - `QuarmItemData` struct for self-found character tracking
  - Various rules (`RuleB(Quarm, ...)` and `RuleI(Quarm, ...)`) for custom behaviors

## Implementation Approach

### Step 1: Create Worktree
Create a git worktree from `showeq-daemon` as the working directory for this work.

### Step 2: Update Protocol Files from EQMacEmu
Copy/update the following files from `EQMacEmu/common/` to `showeq-daemon/src/`:

1. **`src/emu_opcodes.h`** - Update to match EQMacEmu's version (includes chat_oplist.h)
2. **`src/emu_oplist.h`** - Copy from EQMacEmu (auto-generated from DumpOpcodeTable.idc)
3. **`src/chat_oplist.h`** - Chat-specific opcodes (exists in showeq-daemon, may need update)
4. **`src/eq_packet_structs.h`** - Copy packet struct definitions from EQMacEmu
5. **`src/quarm_item_data.h`** - Add Quarm-specific item data support

### Step 3: Update Packet Processing
Modify `src/packetstream.cpp` and `src/packetformat.cpp` to:
- Handle any new/changed net opcodes from EQMacEmu
- Support Quarm-specific packet handling (if any)
- Ensure session tracking works with Quarm's session format

### Step 4: Update Protocol Encoder
Modify `src/protoencoder.cpp` to:
- Handle new opcodes when encoding to protobuf format
- Support Quarm-specific message formats

### Step 5: Update Packet Info Database
Modify `src/packetinfo.cpp` and related files to:
- Register new opcode definitions
- Add payload type definitions for new opcodes

### Step 6: Testing
- Build the daemon with the updated protocol files
- Test against a Quarm server instance
- Verify packet decoding and client streaming works correctly

## Notes

- The `opcodetable.h` in showeq-daemon is auto-generated from EQMacEmu (via DumpOpcodeTable.idc), so copying from EQMacEmu should align versions
- showeq-daemon uses Qt5/Qt6 for the client-facing WebSocket API but the protocol parsing is standalone
- The protobuf schema (`proto/seq/v1/events.proto`) may need updates if Quarm adds new message types not covered by the existing schema
